# Доработки по скринам (интерфейс) с учётом ТЗ «Кадры — люди»

Ниже перечислены изменения в интерфейсе (экранах/макетах), которые нужно внести, чтобы скрины и реализация соответствовали обновлённому ТЗ.

---

## 1. Форма «Добавить руководителя» (скрин: Руководители-добавление.png)

### 1.1. Добавить поле «Категория»

- **Где:** в блоке «Основные детали», в начале формы (логично — перед или сразу после ФИО/организации).
- **Вид:** выпадающий список (справочник).
- **Подпись:** «Категория» или «Группа / Категория».
- **Обязательность:** поле обязательное для заполнения.
- **Значения списка:**  
  Персональный состав (Заместители акима и Рукап), Персональный состав (Акимы районов), Руководители управлений, Руководители департаментов, Депутаты маслихата, Члены общественного совета, Лидеры общественного мнения.

### 1.2. Добавить поле «Партийная принадлежность»

- **Где:** в блоке «Основные детали», после «Должность» / «Организация» или рядом с ними.
- **Вид:** текстовое поле или выпадающий список (если будет справочник партий).
- **Подпись:** «Партийная принадлежность».
- **Логика отображения:** поле **отображается и заполняется только когда в форме выбрана категория «Депутаты маслихата»**. Для всех остальных категорий поле скрыто или неактивно (не заполняется).
- В ТЗ зафиксировано: заполняется только для категории «Депутаты маслихата».

### 1.3. Контакты — одно поле

- **Текущее состояние на скрине:** одно поле «Номер телефона».
- **Требование ТЗ:** контакт один — одно поле «Номер телефона» (без разделения на рабочий/мобильный/приёмная).
- **Действие:** оставить на скрине и в реализации **одно поле** с подписью «Номер телефона» (или «Контактный телефон»). Не добавлять отдельные поля «Рабочий телефон», «Мобильный», «Приёмная».

### 1.4. Вложение (фото)

- **Рекомендация ТЗ:** только изображения (PNG, JPG, JPEG), не PDF; ограничение по размеру (например, до 50 МБ); при необходимости — предупреждение о низком качестве.
- **На скрине:** убрать из подсказки форматы «pdf» для поля фото; указать только допустимые форматы изображений и максимальный размер файла.

## 2. Карточка просмотра/редактирования руководителя (скрин: Руководители-редактирование.png)

### 2.1. Отобразить «Категория»

- В карточке руководителя в блоке основных деталей добавить отображение поля **Категория** (значение из справочника), чтобы было видно, к какой группе отнесён руководитель.

### 2.2. Отобразить «Партийная принадлежность»

- Поле **«Партийная принадлежность»** выводить только если у записи категория «Депутаты маслихата». Иначе не показывать или показывать прочерк/«—».

### 2.3. Контакт — одно поле

- В карточке оставить **одно поле** для телефона (как в ТЗ), без разбиения на рабочий/мобильный/приёмная.

### 2.4. Статус записи

- В карточке отображать **статус** записи: «Активен», «В архиве», «Требует обновления».
- При необходимости добавить действие «В архив» (если его ещё нет в меню строки или в карточке).

---

## 3. Реестр кадров — таблица (скрин: Руководители.png)

### 3.1. Колонка «Категория» (по желанию)

- Для согласованности с ТЗ можно добавить в таблицу колонку **«Категория»**, чтобы было видно группу без открытия карточки. Не обязательно, если достаточно фильтра по категории.

### 3.2. Статусы в таблице

- В колонке «Статус» отображать только три значения: **Активен**, **В архиве**, **Требует обновления**.
- Визуально можно выделять строки со статусом «Требует обновления» (например, цветом или иконкой), так как по ТЗ этот статус выставляется автоматически при отсутствии обновлений более 30 дней.

### 3.3. Выгрузка и выборочная выгрузка

- Кнопки «Выгрузить в PDF» и «Выгрузить в PPTX» оставить.
- **Доработка сценария:** при нажатии на любую из кнопок выгрузки показывать **модальное окно** (или шаг мастера), где пользователь галочками выбирает **категории** для включения в выгрузку. После выбора — генерация файла в выбранном формате.
- На скрине/макете имеет смысл добавить пример такого модального окна: заголовок «Выбор категорий для выгрузки», список категорий с чекбоксами, кнопки «Отмена» и «Выгрузить».

---

## 4. Роли и доступ

- Кнопки выгрузки (PDF/PPTX) и сценарий выборочной выгрузки отображать **только для пользователей с ролью «Администратор протокола»**. Для специалиста кадровой службы эти кнопки скрыты или неактивны (согласно ТЗ).
- На скринах можно сделать два варианта: один — вид от имени специалиста кадров (без кнопок выгрузки), второй — вид от имени администратора протокола (с кнопками выгрузки и модальным окном выбора категорий). Либо подписать на текущем скрине, что экран показан для роли «Администратор протокола».

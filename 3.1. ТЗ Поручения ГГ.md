# ТЕХНИЧЕСКОЕ ЗАДАНИЕ (Brief)

**Название модуля:** Модуль «Поручения ГГ» (Поручения главы государства) для ИС «Протокол» в рамках ОПиСЭР.

**Расположение в системе:** раздел «ОПИСЭР» → **«Поручения ГГ»**. Контент модуля соответствует **стр. 24–28** презентации (слайды Шымкент 2025).

**Источник данных:** данные **не вводятся** в этом модуле. Поручения подтягиваются из **основного модуля системы «Поручения»** (модуль управления поручениями). Дублирование функционала ввода и редактирования поручений не предусматривается.

---

## 1. Цель и назначение

Обеспечить в рамках ОПиСЭР **отображение и выгрузку в слайды** поручений главы государства (ГГ) со статусом **«В работе»**, с последними комментариями исполнителей. Данные берутся только из основного модуля «Поручения»; в модуле «Поручения ГГ» нет собственного реестра и нет ввода/редактирования поручений. Администратор протокола может просматривать список и выгружать слайды стр. 24–28 в PPTX и PDF.

---

## 2. Принцип работы: интеграция с основным модулем «Поручения»

- В системе «Протокол» уже реализован **основной модуль «Поручения»** (управление поручениями).
- Модуль «Поручения ГГ» в ОПиСЭР **не дублирует** этот функционал.
- В разделе ОПиСЭР → «Поручения ГГ» отображаются только те поручения, которые:
  - относятся к категории **«Поручения ГГ»** (глава государства) — по признаку/типу в основном модуле;
  - имеют статус **«В работе»**.
- Для каждого такого поручения подтягиваются данные из основного модуля, в том числе **последние комментарии исполнителей** (по логике основного модуля: последний комментарий по поручению или по исполнителю).
- Редактирование поручений, смена статусов, добавление комментариев выполняются **только в основном модуле «Поручения»**. В «Поручения ГГ» — только просмотр и выгрузка.

---

## 3. Ролевая модель и доступ

**Администратор протокола (глобальный администратор):**
- Имеет доступ к разделу «Поручения ГГ»: **просмотр** списка поручений ГГ со статусом «В работе» с последними комментариями исполнителей.
- Имеет право **выгрузки** слайдов по поручениям ГГ (стр. 24–28) в PPTX или PDF. Выгрузку выполняет только пользователь с ролью **«Администратор протокола»**.

**Остальные роли:**  
Доступ к разделу «Поручения ГГ» и к выгрузке — по правилам, принятым в системе (при необходимости ограничить только администратором протокола; при необходимости дать просмотр другим ролям — уточняется заказчиком).

---

## 4. Данные, подтягиваемые из основного модуля «Поручения»

Список формируется автоматически по критериям:
- тип/категория поручения: **ГГ** (глава государства);
- статус: **В работе**.

Для каждой записи в модуле «Поручения ГГ» отображаются (минимальный набор; итоговый состав колонок согласуется с макетом слайдов 24–28):

| Данные | Источник | Описание |
|--------|----------|----------|
| Поручение (номер, суть/описание) | Основной модуль «Поручения» | Идентификатор и текст поручения. |
| Исполнитель(и) | Основной модуль «Поручения» | Ответственный исполнитель или список. |
| Срок исполнения | Основной модуль «Поручения» | Плановый срок. |
| Статус | Основной модуль «Поручения» | В данном модуле показываются только со статусом «В работе». |
| **Последние комментарии исполнителей** | Основной модуль «Поручения» | Последний по дате комментарий по поручению (или последние комментарии исполнителей — по правилам, принятым в основном модуле). |

Дополнительные поля (дата поручения, документ-основание, контролёр и т.д.) при необходимости берутся из основного модуля и добавляются в отображение и в выгрузку по макету слайдов.

---

## 5. Соответствие слайдам (стр. 24–28) и таблица на экране

Контент слайдов **стр. 24–28** формируется из данных, подтянутых из основного модуля «Поручения» (поручения ГГ со статусом «В работе» + последние комментарии исполнителей).

Точный вид таблицы на слайдах (заголовки колонок, порядок, разбиение по страницам) необходимо согласовать с макетом презентации «Слайды Шымкент 2025», стр. 24–28. Типовая таблица для отображения и выгрузки:

- Номер / суть поручения  
- Исполнитель  
- Срок  
- Статус (В работе)  
- Последний комментарий исполнителя(ей)  

При разнесении по слайдам (например, по одному слайду на поручение или таблица на несколько поручений на странице) логика распределения задаётся по макету (стр. 24–28).

---

## 6. Требования к интерфейсу

### 6.1. Размещение в системе

- В боковом меню раздела «ОПИСЭР» — пункт **«Поручения ГГ»** (кратко: Поручения ГГ).
- При переходе открывается экран с **таблицей** поручений ГГ со статусом «В работе», с колонками согласно п. 4–5 (в т.ч. последние комментарии исполнителей). Данные только для чтения; переход к редактированию — в основной модуль «Поручения» (при необходимости — ссылка/кнопка «Открыть в Поручениях»).

### 6.2. Режим только просмотра

- Нет форм ввода и редактирования поручений в этом разделе.
- Таблица формируется автоматически по данным из основного модуля (фильтр: ГГ, В работе).
- Обновление списка — при открытии раздела или по кнопке «Обновить» (актуальные данные из модуля «Поручения»).

### 6.3. Выгрузка (роль Администратор протокола)

- Кнопки **«Выгрузить в PDF»** и **«Выгрузить в PPTX»** отображаются только у пользователей с ролью **«Администратор протокола»** (или у ролей, которым открыт доступ к выгрузке).
- При выгрузке формируются слайды **стр. 24–28** по образцу презентации; содержимое таблицы(ц) подставляется из текущего набора поручений ГГ со статусом «В работе» и последними комментариями исполнителей.

---

## 7. Требования к функционалу выгрузки

- Выгрузка доступна **только пользователям с ролью «Администратор протокола»** (или по решению заказчика — иным ролям).
- **Образец:** стр. 24–28 презентации «Слайды Шымкент 2025».
- **Форматы:** PPTX (приоритет) и PDF. Содержимое слайдов формируется из данных основного модуля «Поручения» (поручения ГГ, статус «В работе», последние комментарии исполнителей).
- **PPTX:** таблица(ы) и текст на слайде — редактируемые объекты для возможной ручной правки перед печатью.

---

## 8. User Stories (пользовательские истории)

### Эпик 1: Просмотр поручений ГГ (роль: Администратор протокола и др. при наличии доступа)

**US-G1.1 (Просмотр списка):**  
Как администратор протокола, я хочу открыть раздел «Поручения ГГ» и видеть актуальную таблицу поручений главы государства со статусом «В работе» и последними комментариями исполнителей, чтобы не дублировать ввод данных и видеть сводку для отчётности.

**US-G1.2 (Актуальность данных):**  
Как администратор протокола, я хочу, чтобы список в «Поручения ГГ» автоматически подтягивался из основного модуля «Поручения», чтобы при изменении статуса или появлении новых комментариев в основном модуле данные в ОПиСЭР обновлялись без ручного переноса.

### Эпик 2: Выгрузка слайдов (роль: Администратор протокола)

**US-G2.1 (Выгрузка в презентацию):**  
Как администратор протокола, я хочу нажать «Выгрузить в PPTX» (или PDF) и получить слайды стр. 24–28 с таблицей поручений ГГ (в работе) и последними комментариями исполнителей, чтобы использовать их в отчётности без ручного копирования из модуля «Поручения».

**US-G2.2 (Соответствие макету):**  
Как администратор протокола, я хочу, чтобы таблица на слайдах соответствовала макету презентации (стр. 24–28), чтобы отчёт выглядел единообразно.

### Эпик 3: Технические требования (для разработчиков)

**Tech-G1:** Реализовать выборку из основного модуля «Поручения» по признаку «Поручения ГГ» (тип/категория поручения) и по статусу «В работе». Не создавать отдельный реестр поручений в модуле ОПиСЭР.

**Tech-G2:** Реализовать подтягивание для каждого отобранного поручения **последних комментариев исполнителей** (по правилам, принятым в основном модуле: последний комментарий по поручению или по исполнителю, с датой и автором).

**Tech-G3:** Реализовать генерацию слайдов стр. 24–28 в форматах PPTX и PDF на основе текущей выборки (поручения ГГ, «В работе», с последними комментариями). Структуру таблицы на слайдах согласовать с макетом «Слайды Шымкент 2025», стр. 24–28.

**Tech-G4:** Не дублировать в модуле «Поручения ГГ» функции создания, редактирования поручений и добавления комментариев; при необходимости предусмотреть переход из списка «Поручения ГГ» в карточку поручения в основном модуле «Поручения».

## 9. Сводка: источник данных для слайдов 24–28

| Элемент на слайдах (стр. 24–28) | Источник |
|----------------------------------|----------|
| Список поручений | Основной модуль «Поручения»: выборка по типу «ГГ» и статусу «В работе». |
| Номер, суть поручения, исполнитель, срок, статус | Поля поручения в основном модуле «Поручения». |
| Последние комментарии исполнителей | Комментарии к поручению в основном модуле «Поручения» (последние по дате). |

Контент слайдов заполняется **только** из основного модуля «Поручения». В модуле «Поручения ГГ» данные не вводятся и не хранятся отдельно.

Модуль **«СЭР»** (Социально-экономическое развитие), стр. 29–31 презентации. Данные вносит Управление экономики и бюджетного планирования (роль УЭиБП).

## Дальнейшие шаги по модулю 4

Все следующие шаги нужно делать **с учётом обновлённого ТЗ 4.1** и того, что прототип уже меняли. Продолжение следует.

1. **Привести прототип модуля СЭР в соответствие с ТЗ 4.1**
   - Реализовать выбор периода: год отдельно + период (месяц или квартал) отдельно; явно показывать текущий период и период сравнения (например, «январь–сентябрь 2025 г.» / «январь–сентябрь 2024 г.»).
   - Реализовать автоматический расчёт «Изменение» для блоков стр. 29 и 30.
   - Реализовать цветовую индикацию по показателю (красный/зелёный по правилам п. 3.6 ТЗ 4.1).

2. **Блок бюджета (стр. 31) в прототипе**
   - Выбор года плана и периода/даты исполнения (не фиксированные 2025 г. и 01.01.2026).
   - Ввод и хранение данных по всем 15 функциональным группам; расчёт удельного веса; при выгрузке подставлять топ-5 по удельному весу.

3. **Выгрузка (PPTX/PDF)**
   - Генерация слайдов стр. 29–31 по макету с подстановкой данных и цветовой индикацией изменений.
   - На слайд 31 — доходы по перечню ТЗ и затраты как топ-5 функциональных групп по удельному весу.

4. **Согласование с заказчиком**
   - При необходимости уточнить детали по периодам, составу показателей и формату выгрузки после проверки прототипа.

5. **Исторические данные (п. 3.7 ТЗ 4.1)**
   - Реализовать хранение данных по периоду по каждому показателю (без перезаписи других периодов, без журнала изменений).
   - В каждой строке показателя (стр. 29, 30) — выбор прошлого периода из списка сохранённых по этому показателю и отображение сохранённых значений только для чтения по этой строке.

# Прототип ОПиСЭР

Работающий прототип модуля ОПиСЭР для ИС «Протокол»: React (Vite) + TypeScript.

## Запуск

**Если на Mac нет Node.js и npm** — см. файл [УСТАНОВКА_NODE_NPM.md](УСТАНОВКА_NODE_NPM.md) (три способа установки).

После установки Node и npm:

```bash
npm install
npm run dev
```

Откройте в браузере адрес, указанный в терминале (обычно http://localhost:5173).

## Сборка

```bash
npm run build
npm run preview
```

## Структура

- **ОПИСЭР** (меню слева) → 4 подпункта:
  1. **Кадры** — вкладки «Руководители» и «Партия»: реестр, добавление/редактирование в боковой панели, фильтры, выгрузка (мок).
  2. **Карта социальных рисков** — период, сводка (стр. 20), таблица записей (стр. 21–23), сохранение в localStorage.
  3. **Поручения ГГ** — таблица по ТЗ 3.1 и слайдам 24–28 «Слайды Шымкент 2025» (мок из PDF), группировка по протоколу, чекбоксы «В выгрузку» (localStorage), без редактирования поручений; выгрузка (мок).
  4. **СЭР** — три блока (стр. 29, 30, 31), сохранение в localStorage, выгрузка (мок).

Данные руководителей и партий хранятся в `localStorage`. Регистрация и роли не реализованы — один пользователь со всеми правами.
